# Research: Fix Project Modal Centering and Button Visibility

**Branch**: `001-fix-project-modal` | **Date**: 2026-04-08

## Root Cause Analysis

### Finding 1: Framer Motion Overrides CSS Transform

**Decision**: Use Framer Motion's `style` prop for centering (`x: '-50%'`, `y: '-50%'`) instead of CSS `transform: translate(-50%, -50%)`.

**Rationale**: Framer Motion renders all `transform`-related values (scale, translate, rotate) as a single composed `transform` inline style via CSS custom properties (`--motion-translateX`, `--motion-translateY`, etc.). When the CSS class sets `transform: translate(-50%, -50%)`, the inline style generated by Framer Motion's animation overrides it entirely — the centering is lost from the first animation frame.

By moving the centering into Framer Motion's own `style` prop (`x: '-50%'`, `y: '-50%'`), Framer Motion composes both the centering translation and the animation values into the same `transform` string, so they coexist correctly.

**Current broken code** (`ProjectModal.module.css`):
```css
.modal {
  position: fixed;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);  /* ← overridden by Framer Motion */
}
```

**Current broken animation** (`ProjectModal.tsx`):
```tsx
<motion.div
  className={styles.modal}
  initial={{ opacity: 0, scale: 0.95, y: 10 }}
  animate={{ opacity: 1, scale: 1, y: 0 }}
```

**Fix**: Remove `transform` from CSS; pass `x: '-50%'` and `y: '-50%'` as Framer Motion style values; adjust animation to not use `y` offset (which would conflict with the percentage-based y centering):
```tsx
<motion.div
  className={styles.modal}
  style={{ x: '-50%', y: '-50%' }}
  initial={{ opacity: 0, scale: 0.95 }}
  animate={{ opacity: 1, scale: 1 }}
  exit={{ opacity: 0, scale: 0.95 }}
```

**Alternatives considered**:
- Wrap modal in a non-animated centering `div` and animate only the inner content → Works but adds a DOM node.
- Use `translateX`/`translateY` CSS custom properties directly → Fragile, ties implementation to Framer Motion internals.

### Finding 2: Create Button Clipped by `overflow: hidden` at Short Viewports

**Decision**: Make the modal a flex column with a scrollable body and a fixed footer.

**Rationale**: The modal has `overflow: hidden` and `max-height: 90vh`. On short viewports (or when virtual keyboards appear on mobile), the modal content can exceed 90vh and the footer (containing Cancel + Create) gets clipped. Switching to a flex-column layout with `overflow-y: auto` on the body section ensures the footer is always visible while the content scrolls if needed.

**Fix** (CSS):
```css
.modal {
  /* add */
  display: flex;
  flex-direction: column;
}

.body {
  /* add */
  overflow-y: auto;
  flex: 1;
}
```

**Alternatives considered**:
- Increase `max-height` to 95vh → Doesn't fix the structural clipping issue.
- Change `overflow: hidden` to `overflow: visible` → Would break border-radius clipping on backdrop.
