# AI QA Agent

The AI QA Agent is an automated "Self-Healing" CI/CD assistant designed to help developers debug test failures directly within Pull Requests.

## How It Works (Phase 1)

1. **Trigger**: When a Pull Request is opened or updated, the `.github/workflows/qa-agent.yml` GitHub Action triggers.
2. **Execution**: It runs your standard Playwright End-to-End test suite.
3. **Interception**: If the Playwright tests **fail**, the workflow executes `scripts/qa-agent.js` instead of immediately exiting.
4. **Analysis**:
   - The script parses `test-results.json` generated by Playwright.
   - It runs `git diff origin/main...HEAD` to see exactly what code you changed.
   - It packages the failures and the code diff into a prompt.
   - It calls the **Gemini 2.5 Flash API** to perform SDET/QA analysis.
5. **Resolution**: The Gemini API returns an analysis of _why_ the test failed (e.g., application bug or broken test selector) and provides a markdown snippet with a suggested fix. The script then uses the GitHub REST API to post this analysis directly as a comment attached to the PR.

## Setup Instructions

To enable this feature, you must configure the following repository settings securely.

### 1. Add Gemini API Key

You need an active Gemini API Key for the logic to execute.

1. Go to your GitHub Repository -> **Settings**
2. In the left sidebar, click **Secrets and variables** -> **Actions**
3. Create a **New repository secret**:
   - **Name**: `GEMINI_API_KEY`
   - **Secret**: _(Paste your API key here)_

### 2. Configure GitHub Action Permissions

The AI QA Agent needs permission to read the code and write comments on your PRs.

1. Go to your GitHub Repository -> **Settings**
2. In the left sidebar, click **Actions** -> **General**
3. Under **Workflow permissions**, select **Read and write permissions**.
4. Check the box for "Allow GitHub Actions to create and approve pull requests" if you plan to extend this with auto-fixing PRs in the future.
5. Click **Save**.

## Best Practices for Pull Requests

To get the most accurate and helpful debugging assistance from the AI QA Agent, developers should follow these guidelines when submitting Pull Requests (PRs):

1. **Target the correct branch**: The GitHub action is configured to run when PRs are opened against the `main` branch. Ensure your PR is targeting `main` (or update \`.github/workflows/qa-agent.yml\` if your default branch is different).
2. **Keep PRs small and focused**: The QA Agent analyzes the `git diff` of your branch against `main` to determine the root cause of a failing test. A massive PR with thousands of lines changed makes it significantly harder for the AI to pinpoint _which_ specific file caused the failure. Small, atomic PRs lead to highly accurate QA analysis and AI-generated fixes.
3. **Draft PRs for quick checks**: If you suspect a complex change might break the Playwright E2E tests across different browsers, feel free to push your branch and open a "Draft" PR. The CI pipeline will execute, and if it fails, the Agent will analyze it and comment on the drafted PR—effectively giving you automated AI debugging before any formal human review.
4. **Respond to the Agent's Comments**: The AI Agent acts as an automated SDET. If it provides a code snippet to fix an application bug or update a Playwright selector, you can copy the suggested fix, commit it to your branch, and the CI pipeline will automatically run again to verify the solution.

## Local Testing

If you want to test the script logic locally against a failure without pushing to GitHub:

1. Run Playwright to generate the failing report: `CI=true npx playwright test`
2. Set your environment key:
   - Command Line (Windows PowerShell): `$env:GEMINI_API_KEY="your-key-here"`
3. Run the script: `node scripts/qa-agent.js`

_(Note: Because `GITHUB_TOKEN` won't be set locally, it will bypass posting the comment and just print the AI's analysis directly to your terminal)._
